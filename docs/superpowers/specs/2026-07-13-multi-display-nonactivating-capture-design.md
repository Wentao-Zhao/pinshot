# PinShot 多显示器非激活截图设计

## 目标

- 快捷键唤起截图时，为当前连接的每块显示器展示独立覆盖层。
- 抓图期间不激活 PinShot，不让前台 App 的菜单、页签或临时浮层提前关闭。
- 任意显示器完成或取消截图时，结束整次多屏截图会话。

## 根因

当前实现只选择鼠标所在的 `NSScreen`，因此只能在一块显示器上截图。截图开始前还会调用 `NSApp.activate(ignoringOtherApps: true)`，覆盖窗口随后通过 `makeKeyAndOrderFront` 抢占焦点；前台 App 的临时浮层会在真实屏幕图像捕获前因失焦关闭。

## 方案

1. 开始截图时一次性读取 `NSScreen.screens`，为每块显示器记录显示器 ID、尺寸和屏幕对象。
2. 在后台先抓取所有显示器的静态图像。抓取完成前不显示 PinShot 窗口、不激活 PinShot。
3. 回到主线程后，为每块成功抓取的显示器创建一个 `.nonactivatingPanel` 覆盖窗口，并使用 `orderFrontRegardless()` 展示。
4. 覆盖窗口初始不成为 Key Window；用户点击覆盖层后才允许该非激活面板接收键盘操作。
5. 任一覆盖层触发完成、置顶、保存、复制或取消时，统一关闭全部覆盖层并取消后台任务。
6. 截图准备阶段视为会话已开始，重复快捷键不创建第二批任务。

## 输入事件

- 覆盖视图接受非 Key Window 的首次鼠标事件，第一次按下、拖动和抬起即可完成选区。
- 截图会话开始后临时注册无修饰键 `Esc`，无论覆盖窗口是否取得 Key 状态都可立即取消。
- `Esc` 监听沿用 Carbon 全局快捷键机制，不申请辅助功能或输入监控权限。
- 截图会话结束时立即注销 `Esc`，不影响其他 App 的正常键盘输入。
- 多个 Carbon 监听器使用不同 ID 并过滤事件 ID，避免截图快捷键和取消快捷键互相触发。

## 性能与失败处理

- 多屏图像在一个后台任务中顺序抓取，避免主线程卡顿和多块显示器同时抓图造成瞬时资源峰值。
- 单块显示器抓取失败时跳过该屏；全部失败时显示错误提示。
- 保留现有标注、OCR、默认动作和坐标映射逻辑。

## 测试

- 逻辑测试验证多显示器计划保留全部有效显示器，且执行顺序为“全部抓图完成后再展示”。
- 逻辑测试验证空显示器列表不会启动会话。
- 逻辑测试验证首次鼠标事件策略、`Esc` 键定义和两个快捷键 ID 相互隔离。
- 构建与 GUI 烟测验证应用可启动、覆盖层可展示和退出。
